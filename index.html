Selected project

A hands-on platform project built around real DevOps patterns, not toy “hello world” infrastructure.

GitOps Kubernetes Platform

A multi-environment Kubernetes platform using AWS EKS, Terraform/Terragrunt, ArgoCD, ingress-nginx, and External Secrets. The goal was to create separated dev and prod environments, deploy apps through GitOps, and wire secrets securely from AWS into workloads.

The useful part wasn’t just building it. It was debugging the parts that usually break: CRD conflicts, IRSA trust policies, service account annotations, subnet behavior, ingress paths, and ArgoCD routing behind NGINX.

AWS EKS Terraform Terragrunt ArgoCD Helm External Secrets ingress-nginx

Platform design

  • Dedicated VPC per environment
  • Reusable Terraform modules
  • Terragrunt environment separation
  • ArgoCD root-app deployment pattern

Problems solved

  • External Secrets CRD and webhook cleanup
  • IRSA and AWS Secrets Manager access issues
  • EKS node egress and public subnet behavior
  • ArgoCD UI behind path-based ingress

How I think

I care less about pretty diagrams and more about whether the system can be understood, changed, and recovered when something goes sideways.

Infrastructure should serve the product, not become a shrine to complexity.

My usual debugging flow

  • Understand the architecture and the failure surface
  • Check runtime state, not only the code
  • Compare expected behavior against actual behavior
  • Isolate the smallest broken link
  • Fix it in a repeatable, documented, automatable way
